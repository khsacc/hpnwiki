# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
yarn dev       # Generate _meta.json + start TinaCMS dev server + Next.js (localhost:3000)
yarn build     # Generate _meta.json + tinacms build + next build
yarn start     # Serve production build
```

Local dev requires no external services — `MONGO_URI` absent triggers local-filesystem mode automatically.

Admin UI: `http://localhost:3000/admin/index.html`

## Architecture

### Rendering layer (Nextra v2, Pages Router)

- `pages/index.mdx` — site root
- `pages/docs/**/*.mdx` — all wiki content, served at `/docs/*`
- `pages/_app.tsx` — global KaTeX CSS import
- `theme.config.tsx` — Nextra DocsTheme: logo, sidebar, TOC, and the "TinaCMS で編集" edit button (renders only for `pages/docs/` files)

### Sidebar navigation (auto-generated)

`pages/**/_meta.json` files are **never hand-edited and are gitignored**. They are generated by `scripts/generate-meta.mjs` at the start of every `dev` and `build`. The script reads two frontmatter fields from each `.mdx` file:

- `title` — sidebar label
- `sidebarOrder` — sort order (ascending; defaults to 999)

Subdirectory labels and order come from that directory's `index.mdx`. Always set `title` and `sidebarOrder` when creating a new page.

### CMS layer (TinaCMS self-hosted, no TinaCloud)

- `tina/config.ts` — schema with two collections: `TinaUserCollection` (auth) and `doc` (all wiki pages at `pages/docs/`)
- `tina/database.ts` — **must use `export default`** (TinaCMS CLI calls `require(file).default`). Switches between `createLocalDatabase()` and MongoDB-backed `createDatabase()` based on whether `MONGO_URI` is set.
- `pages/api/tina/[...routes].ts` — Next.js API handler wrapping `TinaNodeBackend`; also the production auth handler (Auth.js / `next-auth` v4)
- `netlify/functions/tina.ts` — standalone Netlify Function alternative (not used when `@netlify/plugin-nextjs` converts API routes automatically)

**Local vs production mode** is governed by two flags evaluated identically in both `tina/config.ts` and `tina/database.ts`:

```ts
const isBrowser = typeof window !== 'undefined'
const isLocal = !isBrowser && (process.env.TINA_PUBLIC_IS_LOCAL === 'true' || !process.env.MONGO_URI)
```

- `isBrowser` guard: TinaCMS config is evaluated in the browser too; all server-only logic (env vars, DB) is skipped there.
- `isLocal` is `true` when running server-side with `TINA_PUBLIC_IS_LOCAL=true` OR with `MONGO_URI` unset.

In local mode: no `authProvider`, filesystem DB (`createLocalDatabase()`).  
In production mode: `UsernamePasswordAuthJSProvider`, MongoDB via `mongodb-level` (`createDatabase()`).  
`contentApiUrlOverride: '/api/tina/gql'` is **always set** (required for the statically-built admin UI in both modes).  
Missing `MONGO_URI` in production mode (server-side, non-browser) throws immediately at startup.

### Data flow (production)

```
GitHub (MDX files) ←→ TinaCMS backend (GitHubProvider)
                              ↓
                     MongoDB (content index cache, collection: "tinacms", db: "tinacms")
                              ↓
                    /api/tina/gql  (GraphQL)
                              ↓
                        Next.js pages
```

### Deployment (Netlify)

`netlify.toml` uses `@netlify/plugin-nextjs`, which converts `pages/api/` routes to Netlify Functions automatically. The build command is `yarn build` (which includes the meta generation step).

## Environment variables

See `.env.example`. Key distinction:

| Variable | Local dev | Production |
|---|---|---|
| `MONGO_URI` | unset (triggers local mode) | required |
| `TINA_PUBLIC_IS_LOCAL` | `true` or unset | `false` |
| `NEXTAUTH_SECRET` | optional | required |
| `GITHUB_PERSONAL_ACCESS_TOKEN` | not needed | required (Contents: read/write) |

## Content conventions

- All wiki pages live in `pages/docs/` as `.mdx` files
- Nested sections are subdirectories; each subdirectory must have an `index.mdx`
- `content/docs/` is unused (legacy path from initial setup — ignore it)
- `content/users/index.json` holds the users array; password field must be TinaCMS PBKDF2 format: `{ "value": "<salt_hex><key_hex>", "passwordChangeRequired": false }` — salt=32 bytes, key=512 bytes, iterations=25000, digest=sha256. bcrypt strings do NOT work.
- `__ref__/` contains reference material (old HTML wiki) — not served by Next.js
